Auto-gate now matches eligibility candidates by the program token from `spec.program` (e.g. `us-ny/snap` → `snap`) rather than the leading prefix of the output name. State-namespaced rules like `ny_snap_categorically_eligible` are now correctly gated into `snap_eligible` for `us-ny/snap`, where the previous prefix-only filter silently rejected them. Also adds a minimal-cover step so gated candidates that are already reachable via another gated rule's dependency graph are dropped — this prevents the gate from AND-ing leaves that the rulespec already rolls up via an OR formula. CLI now loads the corpus from `--rulespec-root` flags or `AXIOM_RULESPEC_REPO_ROOTS` so auto-gate has something to gate against (previously the CLI passed an empty CorpusState and auto-gate silently no-op'd).
